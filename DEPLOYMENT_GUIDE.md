# Production Deployment Guide

This guide details the steps to deploy the **AI-Powered Placement Assistant** application to production environments including Docker, Render, and AWS.

---

## 🔑 Required Environment Variables
Ensure the following variables are configured in your production environments:

| Variable | Description | Recommended Production Value |
| :--- | :--- | :--- |
| `GEMINI_API_KEY` | Google Gemini API Authorization | *Your Google AI Studio Key* |
| `DATABASE_URL` | Primary SQL Database Connection string | `postgresql://user:pass@host:5432/dbname` |
| `SECRET_KEY` | JWT token signature string | *Secure random string (e.g. `openssl rand -hex 32`)* |
| `BACKEND_API_URL` | Frontend client target url | `https://api.yourdomain.com` or Render Backend URL |
| `CHROMA_DB_DIR` | Chroma vector db storage folder | `./chroma_db` (persistent disk mount recommended) |

---

## 🛠️ Deploy Option 1: Render (Quickest Cloud Setup)
Render is an excellent option to host the Python FastAPI backend and Streamlit frontend separately.

### Step A: Deploy the Backend (Web Service)
1. Log in to [Render dashboard](https://render.com) and create a **New Web Service**.
2. Link your Git repository.
3. Configure settings:
   - **Environment**: `Python 3` (or choose Docker option to automatically build from `backend/Dockerfile`)
   - **Root Directory**: `backend`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `uvicorn app.main:app --host 0.0.0.0 --port 8000`
4. In the **Environment Variables** tab, add your `GEMINI_API_KEY`, `SECRET_KEY`, and a database URL (you can provision a PostgreSQL instance on Render with a single click and bind the connection string).
5. (Optional) In the **Disks** section, mount a persistent disk at `/app/chroma_db` to ensure your Chroma vector store persists across redeployments.

### Step B: Deploy the Frontend (Web Service)
1. Create a **New Web Service** for the client.
2. Link your Git repository.
3. Configure settings:
   - **Root Directory**: `frontend`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `streamlit run app.py --server.port 10000 --server.address 0.0.0.0`
4. Under **Environment Variables**, set:
   - `BACKEND_API_URL`: The URL generated by Render for your backend service (e.g. `https://placement-backend.onrender.com/api`).
5. Render exposes the Streamlit port automatically.

## 📦 Deploy Option 2: Docker & Docker Compose
For virtual machines (VMs) or bare-metal servers, Docker Compose sets up the entire application stack under a reverse proxy.

1. Clone your repository on your server.
2. Create your `.env` file containing secrets.
3. Install Docker and Docker Compose on the host.
4. Run:
   ```bash
   docker-compose up -d --build
   ```
5. You can mount an Nginx container or setup Certbot on the host to route incoming HTTP traffic on ports `80`/`443` to `http://localhost:8501` (frontend) and `http://localhost:8000` (backend).

## ☁️ Deploy Option 3: Amazon Web Services (AWS)

### Path A: AWS EC2 (Virtual Machine with Docker Compose)
1. Provision an **EC2 t3.medium** instance running Ubuntu Server.
2. Assign an Elastic IP and configure Security Group rules to open ports:
   - `22` (SSH)
   - `80` (HTTP)
   - `443` (HTTPS)
3. SSH into the instance and install Docker / Docker Compose.
4. Pull the repository, add `.env` secrets, and execute `docker-compose up -d`.
5. Install Nginx and configure server blocks to forward requests to ports `8501` and `8000` with automated SSL renewals (via Let's Encrypt Certbot).

### Path B: AWS ECS (Elastic Container Service - Production Grade)
1. Build and push your Docker images to **AWS ECR (Elastic Container Registry)**:
   - `backend` -> `your-aws-account.dkr.ecr.region.amazonaws.com/placement-backend:latest`
   - `frontend` -> `your-aws-account.dkr.ecr.region.amazonaws.com/placement-frontend:latest`
2. Create an **AWS RDS PostgreSQL** Database Instance.
3. Setup an **AWS ECS Fargate Cluster**.
4. Define two ECS Task Definitions:
   - **Backend Task**: Mounts variables `DATABASE_URL` (RDS link), `GEMINI_API_KEY`, etc.
   - **Frontend Task**: Mounts `BACKEND_API_URL` pointing to the Load Balancer DNS of the backend.
5. Provision an **Application Load Balancer (ALB)**:
   - Forward `/api/*` requests to the Backend ECS Target Group.
   - Forward all other traffic (`/*`) to the Frontend ECS Target Group.
